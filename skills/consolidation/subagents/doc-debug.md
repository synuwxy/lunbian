## 分析指令

### lb-doc-owner 日志

对每个 lb-doc-owner 日志目录，读取以下文件：
- execution-log.md
- business-report.md
- infra-report.md

提取以下信息：

1. **文档维护问题**：子代理发现文档与代码不一致的情况
2. **边界识别问题**：子代理无法判断模块归属的情况
3. **待确认事项**：子代理向开发者询问了什么问题
4. **跨层影响**：基础设施变更影响业务层的情况

### debugger 日志

对每个 debugger 日志目录，读取以下文件：
- execution-log.md
- investigation.md
- diagnosis-report.md

提取以下信息：

1. **bug 根因分析**：为什么这里会写一个 bug？根因是什么？
2. **文档/规约拦截点**：哪些文档或规约可以在 bug 发生前拦截？
   - 如果 CONVENTIONS 中有明确规约但被违反 → 规约执行问题
   - 如果 CONVENTIONS 中没有相关规约 → 规约缺失
   - 如果 INDEX 导航不清晰导致位置错误 → 导航问题
3. **根因类型分布**：代码缺陷/约束违规/文档缺失/架构偏差/能力边界
4. **调查效率**：调查路径是否合理，是否有不必要的探索

对每个问题，判断属于哪个类别（文档质量、Skill 设计、能力边界、用户输入、执行偏差），并引用具体证据。
