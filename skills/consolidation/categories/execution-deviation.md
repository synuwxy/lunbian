# 执行偏差问题

## 定义

AI 未遵循 Skill 定义的约束、行为准则或流程，导致执行结果不符合预期。与 Skill 设计不同：Skill 设计正确，但 AI 没有遵守。

## 信号识别

| 信号来源 | 字段 | 含义 |
|---------|------|------|
| ooda-coder | execution-report.md | AI 未遵循路径规范、未遵循行为准则 |
| ooda-act | 自我纠正记录 | AI 因未遵循约束而编译失败 |
| ooda-eval | 交叉验证差异 | AI 的变更超出预期范围 |
| lb-doc-owner | 变更摘要 | 子代理越界操作、未遵循边界约束 |
| debugger | diagnosis-report.md | AI 未遵循调试流程 |
| reviewer | 审查维度结果 | AI 未遵循审查标准 |
| architecture-guard | 违规项详情 | AI 违反架构约束 |

## 判断逻辑

**行为偏差**：
- if AI 未遵循路径规范（使用片段路径）→ 行为偏差
- if AI 越界操作（修改非管辖范围内的文档）→ 行为偏差
- if AI 未遵循汇报原则（写了文件但没汇报关键信息）→ 行为偏差
- if AI 在代码有歧义时自行推断（未询问开发者）→ 行为偏差

**约束违反**：
- if AI 创建了不存在的文档（违反「不创建缺失文档」约束）→ 约束违反
- if AI 修改了项目级文档（违反「不修改项目级文档」约束）→ 约束违反
- if AI 声称文档一致但未提供证据（违反「证据要求」约束）→ 约束违反

**决策偏差**：
- if AI 在应该询问开发者时没有询问 → 决策偏差
- if AI 的决策不符合决策矩阵 → 决策偏差

**区分其他类别**：
- if 问题出在 Skill 设计不合理（约束不合理，AI 无法遵守）→ Skill 设计
- if 问题出在文档不完整（AI 想遵守但信息不足）→ 文档质量
- if 问题出在用户输入不清晰（AI 无法确定正确行为）→ 用户输入
- if 问题出在 AI 能力不足（AI 想遵守但做不到）→ 能力边界

## 映射目标

| 问题类型 | 映射目标 | 演化操作 |
|---------|---------|---------|
| 行为偏差 | Skill 定义/行为准则 | 加强约束、补充行为准则 |
| 约束违反 | Skill 定义/行为准则 | 加强约束、增加检查点 |
| 决策偏差 | 行为准则 | 调整决策矩阵、增加提醒 |
