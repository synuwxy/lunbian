# 文档质量问题

## 定义

文档（INDEX、CONVENTIONS、README）的内容与代码实际不一致，导致 AI 无法正确执行。包括：内容缺失、内容模糊、内容过时。

## 信号识别

| 信号来源 | 字段 | 含义 |
|---------|------|------|
| ooda-scout | 待确认事项 | scout 无法从文档获取必要信息 |
| ooda-forger | 待确认事项 | forger 从文档获取的规约有歧义 |
| ooda-act | 自我纠正记录 | act 因文档信息错误而编译失败 |
| ooda-eval | 交叉验证差异 | 文档描述与代码实际不一致 |
| lb-doc-owner | 变更摘要 | 子代理发现文档与代码不一致 |
| debugger | diagnosis-report.md 根因 | 根因指向文档缺失或过时 |
| reviewer | 审查维度结果 | 审查发现文档规约被违反 |
| architecture-guard | 违规项详情 | 架构约束与代码实际不一致 |

## 判断逻辑

**约束质量问题**（CONVENTIONS）：
- if 代码中存在明确的编码模式，但 CONVENTIONS 未记录 → 规约缺失
- if CONVENTIONS 的描述有多种理解方式，AI 无法确定正确行为 → 规约模糊
- if CONVENTIONS 的描述与代码实际不一致 → 规约过时

**信息质量问题**（INDEX）：
- if AI 无法通过 INDEX 找到目标模块 → 导航缺陷
- if AI 缺少必要的项目信息（业务背景、架构约束等）→ 上下文缺失
- if INDEX 的描述与代码实际不一致 → 上下文过时

**区分其他类别**：
- if 问题出在 AI 没有遵循文档（文档正确，AI 行为错误）→ 执行偏差，不是文档质量
- if 问题出在 Skill 定义的流程有缺陷（文档正确，流程错误）→ Skill 设计，不是文档质量
- if 问题出在用户输入不清晰（文档正确，输入错误）→ 用户输入，不是文档质量

## 映射目标

| 问题类型 | 映射目标 | 演化操作 |
|---------|---------|---------|
| 规约缺失 | CONVENTIONS | 新增规约条款 |
| 规约模糊 | CONVENTIONS | 补充示例、明确边界 |
| 规约过时 | CONVENTIONS | 修正规约条款 |
| 导航缺陷 | INDEX | 补充模块条目、修正路径 |
| 上下文缺失 | INDEX/README | 补充模块描述 |
| 上下文过时 | INDEX/README | 修正模块描述 |
