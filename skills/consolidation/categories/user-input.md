# 用户输入问题

## 定义

开发者的输入（需求描述、指令、配置）存在缺陷，导致 AI 无法正确执行。与文档质量和 Skill 设计不同：文档和 Skill 都正确，但输入本身有问题。

## 信号识别

| 信号来源 | 字段 | 含义 |
|---------|------|------|
| ooda-scout | 待确认事项 | scout 询问需求的具体含义 |
| ooda-forger | 待确认事项 | forger 询问需求的边界和范围 |
| ooda-eval | 交叉验证差异 | 实际变更与用户期望不一致 |
| debugger | 待确认事项 | debugger 询问问题的具体表现和复现步骤 |
| lb-doc-owner | 待确认事项 | 子代理询问模块归属或文档目标 |

## 判断逻辑

**需求模糊**：
- if 多个子代理询问「这个需求的具体含义是什么」→ 需求模糊
- if AI 无法从输入中确定任务范围 → 需求模糊

**需求不完整**：
- if AI 需要额外信息才能执行，但这些信息本应由用户提供 → 需求不完整
- if AI 的输出与用户期望不一致，因为输入缺少关键约束 → 需求不完整

**需求与框架冲突**：
- if 用户的需求与现有文档/规约冲突 → 需求与框架冲突
- if 用户的需求违反架构约束 → 需求与框架冲突

**区分其他类别**：
- if 问题出在文档不完整（输入清晰，文档缺失）→ 文档质量
- if 问题出在 Skill 设计不合理（输入清晰，流程错误）→ Skill 设计
- if 问题出在 AI 没有遵循约束（输入清晰，AI 行为错误）→ 执行偏差
- if 问题出在 AI 能力不足（输入清晰，能力限制）→ 能力边界

## 映射目标

| 问题类型 | 映射目标 | 演化操作 |
|---------|---------|---------|
| 需求模糊 | 无（改善输入习惯） | 提醒开发者提供更清晰的需求描述 |
| 需求不完整 | 无（改善输入习惯） | 提醒开发者提供完整的约束和边界 |
| 需求与框架冲突 | 无（开发者决策） | 提醒开发者需求与框架的冲突，由开发者决定如何处理 |

## 演化建议模板

```markdown
### 用户输入改进建议

| 问题 | 现象 | 建议 | 证据来源 |
|------|------|------|---------|
| [问题类型] | [具体表现] | [改善建议] | [哪些日志体现了这个问题] |
```

**注意**：用户输入问题不映射到框架组成部分，而是提醒开发者改善输入习惯。这是唯一一类「不改框架」的问题。
