# Skill 设计问题

## 定义

Skill 的核心逻辑、流程设计、输出格式、约束定义存在缺陷，导致 AI 无法有效执行。与「执行偏差」不同：执行偏差是 AI 没遵守 Skill，Skill 设计是 Skill 本身有问题。

## 信号识别

| 信号来源 | 字段 | 含义 |
|---------|------|------|
| ooda-coder | execution-log.md 卡点 | 同类 BLOCKED 反复出现 |
| ooda-forger | 待确认事项 | forger 无法制定计划，因为 Skill 的输出格式不完整 |
| ooda-act | 自我纠正记录 | act 因 Skill 的流程缺陷而无法执行 |
| ooda-eval | 待确认事项 | eval 无法判断问题归属，因为 Skill 的分析指引不完整 |
| lb-doc-owner | 变更摘要 | 子代理因 Skill 的约束矛盾而无法执行 |
| debugger | diagnosis-report.md 根因 | 根因指向 Skill 的流程缺陷 |
| reviewer | 审查维度结果 | 审查发现 Skill 的约束不合理 |
| architecture-guard | 违规项详情 | 架构检查规则与 Skill 的流程冲突 |

## 判断逻辑

**流程缺陷**：
- if Skill 的步骤顺序导致依赖问题（前置步骤的输出不完整）→ 流程缺陷
- if Skill 的步骤遗漏关键场景（边界情况未覆盖）→ 流程缺陷
- if Skill 的异常处理不足（错误情况未定义）→ 流程缺陷

**输出格式缺陷**：
- if coordinator 需要的信息不在子代理的返回中 → 输出格式缺陷
- if 子代理的返回格式无法承载必要信息 → 输出格式缺陷

**约束矛盾**：
- if 子代理在两个约束之间左右为难 → 约束矛盾
- if 约束导致行为冲突 → 约束矛盾

**分析指引缺陷**：
- if 子代理在某个场景下「不知道该怎么办」→ 分析指引缺陷
- if 分析指引遗漏关键场景 → 分析指引缺陷

**区分其他类别**：
- if 问题出在 AI 没有遵循 Skill（Skill 正确，AI 行为错误）→ 执行偏差，不是 Skill 设计
- if 问题出在文档内容不完整（Skill 正确，文档错误）→ 文档质量，不是 Skill 设计
- if 问题出在用户输入不清晰（Skill 正确，输入错误）→ 用户输入，不是 Skill 设计

## 映射目标

| 问题类型 | 映射目标 | 演化操作 |
|---------|---------|---------|
| 流程缺陷 | Skill 定义 | 修改步骤顺序、补充场景、增加异常处理 |
| 输出格式缺陷 | Skill 定义 | 修改输出格式、增加字段 |
| 约束矛盾 | Skill 定义/行为准则 | 修改约束、调整决策矩阵 |
| 分析指引缺陷 | Skill 定义 | 补充分析指引、增加场景覆盖 |
