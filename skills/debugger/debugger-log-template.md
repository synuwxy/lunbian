# debugger 执行日志模板

时间戳格式：UTC 紧凑时间戳，格式为 `YYYYMMDDHHmmss`（如 `20260617093045`），每次执行唯一。

日志目录命名：`{UTC紧凑时间戳}-调试`

## 日志目录结构

```
{UTC紧凑时间戳}-调试/
├── investigation.md       # 调查过程详细记录
├── diagnosis-report.md    # 诊断报告（或失败报告）
└── execution-log.md       # 本文件：执行日志汇总
```

## investigation.md 模板

记录调试过程中的每一步操作，用于回溯排查路径。

```markdown
# 调查过程记录

## 问题描述
{用户原始输入，完整记录，不得缩减}

## 排查步骤

### 步骤1：[操作描述]
- 时间：[HH:mm]
- 操作：[读取了哪个文件 / 执行了哪个命令 / 检查了哪个日志]
- 发现：[具体发现了什么]
- 判断：[这个发现意味着什么，是否排除了某个假设]

### 步骤2：[操作描述]
- 时间：[HH:mm]
- 操作：[具体内容]
- 发现：[具体内容]
- 判断：[具体内容]

...（按实际步骤继续记录）

## 排查路径总结
[从问题现象到根因的完整排查路径，包含关键转折点]
```

## execution-log.md 模板

```markdown
# 执行日志

## 任务信息
- 时间戳：{UTC紧凑时间戳（YYYYMMDDHHmmss）}
- 时间：{YYYY-MM-DD HH:mm:ss UTC}
- 用户输入：{完整的用户输入，不得缩减}
- 使用skill：debugger
- 模型：{模型名称}

## 文件链路
- investigation.md：[简述排查路径的关键节点]
- diagnosis-report.md：[简述诊断结论或失败报告摘要]

## 执行过程
- 快速定位：{顺利/卡点，具体描述}
- 根因调查：{顺利/卡点，具体描述}
- 模式分析：{顺利/卡点，具体描述}
- 假设验证：{顺利/卡点，具体描述}

## 调试纪要
- 问题：[简述]
- 根因：[简述]
- 根因类型：[代码缺陷/约束违规/文档缺失/架构偏差/能力边界]
- 调查耗时：[预估]

## 卡点纪要
[只记录问题，顺利的不记录。没有则写"无"]
```
